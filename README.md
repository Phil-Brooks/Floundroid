## 🚀 Latest Stable Release: v0.4.2 (Types Refactor & Optimisation)

This release delivers the largest internal cleanup since Stage 1, focusing on **type‑safety**, **bitboard correctness**, and **UCI stability**.  
The refactor eliminates legacy abstractions, simplifies the move pipeline, and fixes long‑standing edge‑case bugs.

### **Key Improvements in v0.4.2**
- **Complete Types Refactor**
  - Unified `Square`, `File`, `Rank`, and `Move` representations  
  - Stronger invariants and simpler bitboard access  
  - Faster and safer move decoding  
- **Correct UCI Move Application**
  - Board‑aware `fromUci`  
  - No more desync between internal board and GUI  
- **Promotion & Capture Fixes**
  - Eliminated the “quiet promotion” bug  
  - Correct handling of all special moves  
- **Stability Improvements**
  - Zero illegal moves generated by Floundroid  
  - Clean, deterministic search behaviour

## 📊 Calibration & Rating (Updated for v0.4.2 vs Cinnamon 2.0)

Floundroid v0.4.2 was tested against **Cinnamon 2.0** over 100 games at `tc=30`.

### **v0.4.2 Performance vs Cinnamon 2.0**

- **Final Score:** `8 – 85 – 7` (11.5%)  
- **Elo Difference:** `–354 ± 105`  
- **Draw Ratio:** 7%  
- **LOS:** 0.0%

### **Breakdown**
- **As White:**  
  `5 – 42 – 3` (13.0%)  
- **As Black:**  
  `3 – 43 – 4` (10.0%)

### **Interpretation**

Compared to v0.4.1:

- **Strength increased by ~80 Elo**  
- **Wins doubled (4 → 8)**  
- **Search stability improved dramatically**  
- **Zero illegal moves from Floundroid**  
- **Only one illegal move in the entire match — from Cinnamon**

This confirms the refactor was a **major structural success**:

- Move generation is now correct  
- UCI integration is correct  
- Board state remains consistent  
- Hashing and repetition tracking are stable  

Floundroid is now ready for the next wave of evaluation improvements.
